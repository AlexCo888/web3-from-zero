0. Connecting to a Blockchain Network

This tutorial will start from zero, which means that the first lessons don't require you to install anything. You don't need a wallet or dev environment; you just need your browser and basic JavaScript skills.

Note: If you don't have JavaScript skills yet, because you're just started your developer journey, or because you come from a different programming language, I'd recommend FreeCodeCamp. They offer hundreds of hours of JavaScript courses for beginners for free!

Connection Modes

Coming from Web2, you probably know how to load your data from storage into a frontend. You connect to a backend via an HTTP API, and this API, in turn, loads data from storage like a database.

The same is true for Web3. If you don't have a crypto wallet, you need an HTTP API to connect to a blockchain network. This API is called a gateway. It accepts requests from a browser via HTTP and routes them to a blockchain network. You can only read data from the blockchain with this connection mode.

If you have a crypto wallet, you can make it accessible with a browser extension for a frontend. In the case of Ethereum, such an extension will inject a global ethereum object into your frontend that you can use to interact with your wallet. You can use this connection mode to read and write blockchain data, like sending transactions.

If you have a wallet installed and the extension set up, it will connect to the blockchain network via an HTTP gateway, so the browser doesn't have to interact with the gateway directly.

Using Ethers.js

Ethers.js is a library that helps with connecting to the Ethereum network. When using Ethers.js, you don't have to care if you connect directly via an HTTP gateway or use a wallet extension. After you establish a connection, you can call the same functions.

Figure 2 shows how Ethers.js fits between your frontend and the Ethereum network.

Ethers.js comes in the form of an ES-module on the NPM package registry. This means, you can import it directly in the browser from the Unpkg CDN.

Listing 1 shows how you import the ethers package into your code.

import {ethers} from "//unpkg.com/ethers@5.5.1/dist/ethers.esm.min.js"
Listing 1: Importing the Ethers.js package

Connecting to the Ethereum Network

To connect to the Ethereum network, you have to create a provider. Providers are the abstraction Ethers.js uses to normalize access via wallet and HTTP gateway.

You can create such an provider by calling the getDefaultProvider method of the ethers object you imported before.

Note: This tutorial features an interactive editor. Here you can write JavaScript code and execute it with the press of a button. The Ethers.js library is imported for you and you can us await for asynchronous requests. It also has a print function you can use for debugging.

The first example is already complete to show you how things work. Just press the Execute button and check the result.

const provider = ethers.getDefaultProvider() print(provider)

Let's try it yourself!

To read data from the Etherum network, you can call the getBlock method of the ethers object. It returns a promise that resolves to an object that contains the block's data, so you need to await it. After you got the block, you can display its content with the print function.

const provider = ethers.getDefaultProvider() // Write your code here!

const block = await provider.getBlock() print(block)

If everything went correctly, you see the block data in the results. This data is all blockchain-specific information that doesn't tell us anything interesting yet. At least, you see that the connection worked and something came back!

The Ethers.js library comes pre-configured with a public HTTP gateway you can use for free to read data from the Ethereum blockchain. This gateway has a request frequency limit, so it can happen that you don't get a response if you send too many requests. But for this tutorial, it should be enough.

Conclusion
